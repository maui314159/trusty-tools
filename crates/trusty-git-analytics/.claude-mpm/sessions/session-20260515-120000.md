## Project Overview

**trusty-git-analytics** is a Rust port of the Python `gitflow-analytics` tool—a developer productivity analytics CLI with full git extraction, commit classification, and reporting capabilities.

**Architecture**:
- Single `tga` crate (library + binary)
- Three-stage pipeline: collect → classify → report
- SQLite backend with migration runner
- YAML configuration
- GitHub Actions CI/CD

## Technical Reminders

### Critical (🔴)
- Always use WAL mode: `PRAGMA journal_mode=WAL` on DB opens
- Use `anyhow::Result` in `main.rs`, `thiserror` in libraries
- All public APIs need doc comments (`///`)
- No `unwrap()` in library code

### Important (🟡)
- `cargo clippy -- -D warnings` must pass before merge
- `cargo fmt --check` must pass in CI
- All 4-tier classification cascade + LLM fallback

### Database
- SQLite schema in `src/core/db/sql/` (v1–v18 migrations)
- Migration runner applies on startup
- Schema reference: `docs/requirements/database-schema.md`

### CLI Structure
- Binary: `tga` from `src/main.rs`
- Subcommands: `analyze`, `collect`, `classify`, `report`
- Config via YAML (same as Python version)
