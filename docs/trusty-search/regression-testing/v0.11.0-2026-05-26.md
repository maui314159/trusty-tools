# Trusty-Search v0.11.0 Benchmark — May 26, 2026

**Date**: 2026-05-26
**Version**: v0.11.0 ([#152](https://github.com/bobmatnyc/trusty-tools/issues/152) PROVENANCE-ONLY — remove community_cohesion + Louvain)
**Tracking Issue**: [#129](https://github.com/bobmatnyc/trusty-tools/issues/129)
**Related**: [#145](https://github.com/bobmatnyc/trusty-tools/issues/145) (Stage 3 KG retrieval-lift decision), [#152](https://github.com/bobmatnyc/trusty-tools/issues/152) (this release)

**Primary question**: Does removing the community_cohesion blend and Louvain
community detection break any existing queries? Does the symbol graph (KG
provenance navigation) continue to function correctly after the removal?

## Change Summary

Issue #145 empirical data: `search_kg` LOST Hit@1 by 16.7 pp vs
`search_semantic` (7/18 vs 10/18) on KG-targeted queries. The community
cohesion bonus was hurting rather than helping result quality.

### Deleted

- `src/core/community.rs` — Louvain implementation (673 lines)
- `src/core/indexer/graph_score.rs` — GraphScorer / centrality bonus (309 lines)
- `SearchAppState::graph_scorer()` + `invalidate_graph_scorer()`
- `GraphScorerCache` type alias + `spawn_community_detection` reindex task
- `CodeChunk::community_id` field
- `GET /indexes/:id/communities` endpoint
- `GET /indexes/:id/communities/:symbol` endpoint
- `meta.graph_scoring` + `meta.community_cohesion` from search response JSON

### Preserved

- `SymbolGraph` (petgraph call graph) — powers `get_call_chain` + `search_kg`
- `KG_COMMUNITIES_TABLE` / `kg_symbol_community` redb table definitions —
  kept for migration tolerance; no longer written or read by search path
- All 4 per-lane MCP tools (`search_lexical`, `search_semantic`, `search_kg`,
  `search_all`)

## Migration Notes

- Callers reading `meta.graph_scoring` or `meta.community_cohesion` from
  search responses must remove those references — fields are gone.
- Callers polling `GET /indexes/:id/communities` must remove those calls.
- `CodeChunk` JSON with `community_id` present is tolerated (ignored on
  deserialise via `#[serde(default)]`). No schema migration required.
- Old redb databases with community tables work without migration — the
  table definitions remain in `corpus.rs` for compatibility; data is simply
  not read by the active search path.
