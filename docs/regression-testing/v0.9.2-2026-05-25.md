## #122 Change Summary

Commit 19fef0b adds `is_function_definition_chunk_type()` and extends `apply_score_adjustments` to boost `Function`/`Method` chunks (in addition to the existing `Struct`/`Enum`/`Class`/`Trait`/`TypeAlias` boost from #117) when the chunk's `function_name` contains a Definition-intent query token. `Constant`, `Statement`, `Docstring`, and `Code` chunk types are **explicitly excluded** — the motivation is to avoid boosting string-literal occurrences (e.g. a JSON tool descriptor that contains a function name as a text token).

## Q04 (BRUSILOV_EPOCH) Before/After Analysis

### Key Finding: #122 Did NOT Fix Q04 — Expected by Design

**Q04 remains a universal miss across all three modes.** `src/constants.rs` does not appear in the top 10 in any mode.

**Root cause confirmed**: `BRUSILOV_EPOCH` is defined as a `pub const` in `constants.rs`. The tree-sitter chunker classifies the entire `constants.rs` file as a single chunk with `chunk_type = Code` (the file was chunked as a monolithic code block; individual `pub const` items were not split out as `Constant`-typed chunks — they are embedded in the whole-file chunk with no `function_name` set).

The #122 boost fires only for `Function` and `Method` chunk types. `Code` (whole-file fallback), `Constant`, and `Class` chunks without a matching `function_name` are not boosted. Since `constants.rs` has `chunk_type=Code` and `function_name=null`, neither the #117 struct boost nor the #122 function boost applies. The boost mechanism correctly bypasses it, but the result is that `constants.rs` stays unranked.

### Two Compound Problems for Q04

**Problem 1 — Intent classification miss**: `BRUSILOV_EPOCH` is classified as `Unknown` intent, not `Definition`. The `QueryClassifier` recognizes PascalCase and snake_case identifiers as Definition signals (fixes from #119), but `SCREAMING_SNAKE_CASE` constants are not in the classifier's pattern set. Because intent is `Unknown`, `struct_boost_tokens` is empty, and neither the #117 nor the #122 boost fires even for chunks that do have a matching `function_name`.

**Problem 2 — Constant chunk has no `function_name`**: Even if intent were correctly classified as `Definition`, `constants.rs` would still be skipped by both boosts because the chunk has `function_name=null`. The #122 design comment acknowledges that the boost requires a non-null `function_name` for the token-substring check. `pub const` declarations are not currently promoted to named chunks with their own `function_name` in the tree-sitter AST chunker.

**Phase 2 recommendation**: A standalone `Constant` chunk type with `function_name = "BRUSILOV_EPOCH"` (populated by the Rust chunker when it encounters `pub const NAME`) would allow the boost to fire. Alternatively, extending the `QueryClassifier` to recognize `SCREAMING_SNAKE` as a `Definition` intent without requiring the boost would at least enable the existing RRF + inject_entity_exact_match path to route correctly.

## Caveats

1. **v0.9.2 is the first version snapshot measured against the clean synthetic corpus.** The v0.9.1 v2 baseline (documented in `synthetic-corpus-baseline-2026-05-25.md`) and this v0.9.2 snapshot use identical corpus, harness, and query set — so the delta is a clean apples-to-apples comparison. Earlier version snapshots (v0.8.x, v0.9.0) used the circular-biased trusty-tools corpus and are not directly comparable.
2. **Warm-boot RSS difference** (v0.9.1: 3830 MB, v0.9.2: 1047 MB after 100s). The v0.9.1 daemon had 5710s uptime at the time of measurement (all HNSW structures paged in from prior queries). The v0.9.2 daemon was freshly started — RSS reflects only the structures that have been demand-paged during the 100s observation window. Not a regression; different observation timing.
3. **Q04 is a known gap.** The failure is structurally understood and documented above as Phase 2 work. It is not a regression introduced by v0.9.2.
4. **Single run.** p50 latency estimates; tail latency not characterized.

## Follow-Ups

- **Phase 2 / Q04 fix**: Extend Rust tree-sitter chunker to emit individual `Constant`-typed chunks with `function_name` populated from the `pub const NAME` declaration, and/or extend `QueryClassifier` to recognize `SCREAMING_SNAKE` identifiers as `Definition` intent. Track in a new issue.
- **Q11 H@1 miss** (CHANGELOG outranks README for "Glyphwarpen Observatory" query): ranking within `text` mode. Future refinement.
- **Q19 H@1 miss** (pipeline.rs at rank 1 instead of wolfram/inventory.rs): the query "accumulate pipeline metrics and expose read-only summary" vectorially resembles `pipeline.rs` because of the word "pipeline". A harder semantic target.
- **format_in_format_args clippy warning** (noted by sibling engineer): a stray `format!("{}", x)` that should be `x.to_string()` in the benchmark harness or tests — file a separate small bug once the exact location is confirmed.
